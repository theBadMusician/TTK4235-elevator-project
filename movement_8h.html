Detailed Description

Handles elevator routing, stopping logic, and direction calculation.

This module implements the core elevator algorithm (the LOOK algorithm). It determines when the elevator should stop, which direction it should travel next based on active orders, and how to recover if halted between floors.

void movement_directionHandler ( ElevatorState *  elev)

Calculates and applies the next motor direction based on active orders.

  • Parameters
    [in,out]elevPointer to the central elevator state structure.
  • Checks for remaining requests in the current direction of travel. If orders remain, it continues in that direction. If the path is clear, it reverses direction to serve orders behind it. If the entire queue is empty, it stops the motor and transitions the system into STATE_IDLE.

◆ movement_recoveryHandler()

bool movement_recoveryHandler ( ElevatorState *  elev)

Determines the initial direction if the elevator is halted between floors.

  • Parameters
    [in,out]elevPointer to the central elevator state structure.
    Returns
    true if a recovery was necessary and handled.
    false if the elevator is not in a recovery state.
  • If the elevator is stopped between floors (sensor reads -1) and a new order is placed, this function uses the previous direction and previous floor to calculate whether it is "hovering" above or below a floor. It then routes the elevator safely to the nearest requested floor without skipping orders.

◆ movement_stopHandler()

bool movement_stopHandler ( ElevatorState *  elev)

Evaluates whether the elevator should halt at the current floor.

  • Parameters
    [in]elevPointer to the central elevator state structure.
    Returns
    true if the elevator should stop at this floor.
    false if the elevator should bypass this floor.
  • Implements the core stopping logic: Always stops for cabin calls. If moving UP, it stops for UP hall calls, or DOWN hall calls if it is the highest active request. If moving DOWN, it stops for DOWN hall calls, or UP hall calls if it is the lowest active request.
